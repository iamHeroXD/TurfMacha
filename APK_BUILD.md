# TurfMacha — Android APK Build Guide

TurfMacha is now an **app-first** product wrapped as a native Android app with
[Capacitor](https://capacitorjs.com/). The native shell (`android/`) loads the
live, hosted Next.js app inside a WebView, so all server features — API routes,
payments, SSR, Supabase auth — keep working with **no backend duplication**.

The app opens on the **login screen**; everything else (dashboard, turf
browsing, bookings, wallet, profile) is reachable only after signing in.

---

## How it's wired

| Piece | Where |
| --- | --- |
| Native Android project | `android/` (generated, committable) |
| Capacitor config | `capacitor.config.ts` |
| Hosted app URL the APK loads | `server.url` in `capacitor.config.ts` (default `https://turfmacha.vercel.app`) |
| Offline fallback splash | `www/index.html` |
| App icon / splash source | `assets/icon.png`, `assets/splash.png` |

To target a different deployment, set `CAP_SERVER_URL` before syncing:

```bash
CAP_SERVER_URL=https://staging.turfmacha.app npx cap sync android
```

## Prerequisites (one-time, on the build machine)

These are **not** installed in this repo's container — install them locally:

1. **JDK 21** (Temurin/Adoptium recommended — Capacitor 8 requires it). Verify: `java -version`
2. **Android Studio** (includes the Android SDK + command-line tools).
   - In Android Studio → *SDK Manager*, install **Android SDK Platform 34+**
     and **Android SDK Build-Tools**.
3. Set `ANDROID_HOME` (or `ANDROID_SDK_ROOT`) to your SDK path, e.g.
   `C:\Users\<you>\AppData\Local\Android\Sdk`.

## Build a debug APK (fastest path)

```bash
# 1. make sure the native project has the latest config/assets
npm run cap:sync

# 2. build the APK with Gradle
npm run android:apk
```

The APK lands at:

```
android/app/build/outputs/apk/debug/app-debug.apk
```

Copy it to an Android phone and install (enable *Install unknown apps*).

> Prefer a GUI? Run `npm run android:open` to open the project in Android
> Studio, then **Run ▶** onto a device/emulator, or
> *Build → Build Bundle(s) / APK(s) → Build APK(s)*.

## Build a signed release APK (for distribution)

1. Create a keystore (one-time):

   ```bash
   keytool -genkey -v -keystore turfmacha.keystore \
     -alias turfmacha -keyalg RSA -keysize 2048 -validity 10000
   ```

2. Create `android/keystore.properties` (do **not** commit it):

   ```properties
   storeFile=../../turfmacha.keystore
   storePassword=YOUR_STORE_PASSWORD
   keyAlias=turfmacha
   keyPassword=YOUR_KEY_PASSWORD
   ```

3. Reference it in `android/app/build.gradle` under `signingConfigs` /
   `buildTypes.release` (see the Capacitor docs:
   https://capacitorjs.com/docs/android/deploying-to-google-play).

4. Build:

   ```bash
   npm run android:apk:release
   # -> android/app/build/outputs/apk/release/app-release.apk
   ```

   For Google Play, build an **AAB** instead: `cd android && gradlew.bat bundleRelease`.

## Updating the app

Because the UI is served from `server.url`, **most changes ship by simply
deploying the web app** — users get them on next launch, no new APK needed.

Rebuild the APK only when you change:
- the app id / name / icon / splash,
- the `server.url`,
- native plugins or Android config.

After such changes: `npm run cap:sync` → rebuild.

---

## App identity

- **App ID:** `app.turfmacha.android`
- **App name:** `TurfMacha`

Change these in `capacitor.config.ts` (`appId` / `appName`), then re-run
`npx cap sync android`. Note: changing `appId` after publishing creates a
*new* Play Store listing.

## Notes / gotchas

- **Google OAuth in a WebView:** Google blocks its OAuth flow inside embedded
  WebViews. Email/password sign-in works as-is. For native Google sign-in,
  add `@capacitor/browser` or a native Google auth plugin later.
- **Deep links / push:** not configured yet — add `@capacitor/app` deep links
  and FCM via a Capacitor push plugin when needed.
- The `android/` folder is generated by Capacitor but safe to commit so the
  build is reproducible.
